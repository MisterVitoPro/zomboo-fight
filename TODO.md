## 11. TECHNICAL & ARCHITECTURE

- [ ] **[HIGH] Upgrade to 60 FPS** -- Change `FPS = 60`, normalize all speed/timer constants by 2. Existing `dt` pipeline supports this. Smoother movement and aiming.
- [ ] **[HIGH] Centralized Config File** -- Extract all balance values (HP, speed, damage, spawn rates, probabilities) into `config.json`. Enables rapid iteration without touching code. Foundation for difficulty settings and modding.
- [ ] **[HIGH] Game State Machine** -- Replace boolean-stitched menu/game/gameover flow with `MenuState`, `PlayingState`, `PausedState`, `GameOverState` classes. Enables pause screen and clean transitions.
- [ ] **[MED] Resolution-Independent Rendering** -- Render to 800x600 logical surface, `smoothscale` to display size. Enables fullscreen and widescreen without changing sprite math.
- [ ] **[MED] Spatial Hash for Collision** -- Grid-based spatial hash (64px cells) instead of brute-force `spritecollide`. Dramatically reduces O(n^2) checks with 50+ zombies.
- [ ] **[MED] Sound Manager with Channel Pooling** -- Centralized `SoundManager` with pre-loaded sounds, managed channels, volume categories (SFX, music). Prevents channel exhaustion during heavy combat.
- [ ] **[MED] Object Pooling for Bullets** -- Pre-allocate 100+ bullet objects, reset on "spawn", return to pool on `kill()`. Eliminates per-shot allocation overhead. Critical for 60 FPS with MP5/flamethrower.
- [ ] **[MED] Lightweight Particle System** -- Replace full `GameObject` subclasses (HitSplat, Explosion) with pooled particles: simple rects with velocity, lifetime, fade. Reduces GC pressure and enables richer effects.
- [ ] **[LOW] Sprite Group Consolidation** -- Replace 13 separate groups with single `LayeredUpdates`. Dynamic z-ordering, simplified update/draw, fixes module-level singleton issue.
- [ ] **[LOW] Component-Based Entity Architecture** -- Refactor `GameObject` monolith into composable components (Movement, Animation, Health, Collision). Makes new entity types trivial to add.
- [ ] **[LOW] Save/Load System** -- `serialize()`/`deserialize()` on entities, JSON snapshot of all sprite groups. Enables mid-game saves and replay foundation.
- [ ] **[LOW] Dirty-Rect Rendering** -- Switch to `LayeredDirty` with `DirtySprite`. Only redraw changed rectangles. 50-70% render cost reduction when action is sparse.
